Penjelasan Struktur Backend Modular

Proyek backend ini telah direfaktor untuk mengikuti struktur yang lebih modular dan terukur. 
Tujuannya adalah untuk memisahkan setiap bagian dari aplikasi agar lebih mudah dikelola, dikembangkan, dan di-debug.

Penjelasan Direktori:

- `src/`: Direktori ini adalah direktori utama dari aplikasi backend Anda.

- `src/controllers/`: Direktori ini berisi file-file yang bertanggung jawab untuk menangani logika bisnis dari aplikasi Anda.
  - `wisataController.js`: File ini berisi semua fungsi yang menangani permintaan terkait data wisata, seperti mengambil semua data, mengambil data berdasarkan ID, membuat, memperbarui, dan menghapus data.

- `src/data/`: Direktori ini berisi data statis atau file yang berhubungan dengan data.
  - `wisata.js`: File ini berisi array data wisata yang sebelumnya ada di `server.js`.

- `src/routes/`: Direktori ini berisi file-file yang mendefinisikan rute atau endpoint API dari aplikasi Anda.
  - `wisataRoutes.js`: File ini mendefinisikan semua rute yang terkait dengan data wisata (misalnya, `/api/data`, `/api/data/:id`) dan menghubungkannya dengan fungsi-fungsi yang ada di `wisataController.js`. File ini juga berisi dokumentasi Swagger/OpenAPI untuk setiap endpoint.

Penjelasan File Utama:

- `src/server.js`: Ini adalah file utama dari aplikasi Anda. Tanggung jawabnya sekarang adalah untuk:
  - Menginisialisasi server Express.
  - Menerapkan middleware (seperti `cors` dan `express.json`).
  - Mengkonfigurasi Swagger/OpenAPI.
  - Menghubungkan rute-rute dari direktori `routes` ke aplikasi utama.
  - Menjalankan server.

Keuntungan dari Struktur Modular Ini:

- **Keterbacaan yang Lebih Baik**: Kode lebih mudah dibaca dan dipahami karena setiap file memiliki tanggung jawab yang jelas.
- **Perawatan yang Mudah**: Ketika Anda perlu mengubah logika bisnis, Anda tahu persis harus membuka file di direktori `controllers`. Jika Anda perlu menambahkan atau mengubah endpoint, Anda hanya perlu mengubah file di direktori `routes`.
- **Skalabilitas**: Struktur ini memudahkan untuk menambahkan fitur-fitur baru. Misalnya, jika Anda ingin menambahkan fungsionalitas pengguna, Anda hanya perlu membuat `userController.js`, `userRoutes.js`, dan mungkin `userData.js`.
- **Pengujian yang Lebih Mudah**: Memisahkan logika bisnis ke dalam controller memudahkan untuk melakukan unit testing pada setiap fungsi.
